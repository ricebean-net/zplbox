Description

ZplBox is based on an embedded Chromium, rendering your web content as PNG. So, you can leverage the full web technology stack to create your labels. You can submit your web content via a file or by referencing it using an external URL (preferred).

The conversion of PDF documents to PNG is based on Apache PDFBox

Once rendered as PNG, your content will be converted into a monochrome image and encoded in a ZPL native ASCII hexadecimal string. The final ZPL label consists of only the ASCII hexadecimal string embedded in a Graphic Field (^GF) and will sent back in the response or directly forwarded to the label printer (ZPL Print Server).

This approach will allow you new possibilities, which are nearly impossible with native ZPL, including:

  • Images: Embed and scale images flexibly using default web technologies.
  • Fonts: You can use any fonts supported by web frameworks.
  • Text / Typography: You can use any methods you use in web design.
  • Special Characters: No restrictions regarding special characters anymore.
  • ...

ZPL Print Server

ZplBox can function as a local ZPL print server, allowing you to hide all ZPL-specific logic from your developers by abstracting it through standard web interfaces like REST or HTML. By using one of the "**/print/{TCP_ADDRESS}" endpoints, ZplBox generates ZPL labels from the provided input and forwards them directly to the specified TCP address.

Quick Start

1. Start ZplBox as Docker container

ZplBox is designed as a Docker container what can be started using the following command: 
docker run -p 8080:8080 ghcr.io/meixxi/zplbox:latest

2. Open ZplBox UI

ZplBox is delivered with an integrated user interface (UI) allowing you to get used to ZplBox and its API. The UI can be used to manually convert HTML to ZPL and PDF to ZPL and to send the generated ZPL labels to a printer. The UI is available at http://localhost:8080.

API Reference

Example

Here is a first curl example of submitting an UPS test label to the API. You will find more, automatically generated, examples in the ZplBox user interface. 
curl --request POST \
     --url http://localhost:8080/v1/html2zpl \
     --header 'content-type: application/json' \
     --data '{ "url":"http://localhost:8080/labels/ups-example.html", "widthPts":812, "heightPts":1624 }'
NOTE: You can copy the generated ZPL Code to https://labelary.com/viewer.html to get a visual representation of the output.
